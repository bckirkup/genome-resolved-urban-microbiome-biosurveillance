# **Collection and Cleaning Submodule**
### *Automated Aggregation and Downstream Preparation for Genome-Resolved Biosurveillance*

This submodule provides the **final stage of the bioinformatics workflow**, integrating outputs from genome assembly, annotation, and abundance quantification into unified, analysis-ready datasets.  
It performs **file aggregation, functional filtering, species-level mapping**, and **data standardization** for ecological and machine learning analyses.

---

##  **Overview**

This module automates:
- Global merging of MAG-level abundance tables  
- Mapping of annotated MAGs to GTDB species taxonomy  
- Selection of genomes carrying **AMR (CARD)** or **virulence (VFDB)** genes  
- Labeling of species identifiers for unified abundance matrices  
- Generation of downstream-ready merged datasets for ecological modeling  

All scripts are **SLURM-compatible**, modular, and designed for reproducibility in large-scale biosurveillance analyses.

## **Workflow Summary**

### **1 Global File Processing**
**Script:** `01_Global_file_abundance_processsing.sh`  
Processes per-sample abundance outputs (TPM, RPKM, coverage fraction) generated by **CoverM** for all environments.  
Normalizes file formats, validates headers, and ensures consistent delimiter structure for merging.

*Output:** Cleaned abundance tables (`*_processed_abundance.tsv`)

---

### **2 Merge Abundances Across Environments**
**Script:** `02_Merge_abundances.sh`  
Combines environment-specific processed files into a single **global abundance matrix**, merging samples from hospital sewage, hospital environment, ambulances, and public transport.  
Handles missing data and zero-padding for non-detected species.

*Output:** `Merged_abundance_matrix_global.tsv`

---

### **3️ Map Dereplicated MAGs to GTDB Species**
**Script:** `03_Mapping_to_species.sh`  
Links dereplicated MAGs to GTDB-Tk taxonomy assignments (R226 release).  
This ensures that downstream abundance and functional annotations are aggregated at **species level**, rather than genome level.

*Output:** `MAG_to_species_mapping.tsv`

---

### **4️ Select Genomes Carrying ARG or VF Determinants**
**Script:** `04_Selecting_ARG_VF_genomes.sh`  
Filters the dereplicated MAG catalog to retain only genomes annotated with at least one **antimicrobial resistance (ARG)** or **virulence factor (VF)** gene.  
This functional filtering step generates a pathogen-focused subset of the urban microbiome for ecological analysis.

**Output:**  
- `Selected_ARG_VF_MAGs.tsv`  
- `ARG_VF_abundance_matrix.tsv`

---

### **5️ Label and Standardize Species Metadata**
**Script:** `05_labelling_species.sh`  
Applies unified species labeling conventions across merged abundance and annotation files.  
Automatically handles unresolved species names by appending `spp.` suffixes at genus level  
Standardizes species IDs for reproducibility and machine learning compatibility.

*Output Directory:**
```
merged_outputs/
├── MAG_to_species_mapping.tsv
├── Labeled_species_abundance.tsv
└── ARG_VF_selected_genomes.tsv
```
